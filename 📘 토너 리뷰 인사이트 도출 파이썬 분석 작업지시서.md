📘 토너 리뷰 인사이트 도출 파이썬 분석 작업지시서

(10대 핵심 인사이트 × 월별 트렌드 결합)

0️⃣ 목적 정의 (중요)

목표

기존에 정의한 핵심 인사이트 10가지를

월별 트렌드 변화와 결합하여

“말이 되는 인사이트”가 아니라 데이터로 증명 가능한 인사이트를 도출한다.

최종 산출물

월별 × 속성 × 감성 지표 테이블

월별 인사이트 후보 테이블

대시보드 / 리포트용 CSV

1️⃣ 입력 데이터 명세
📥 입력 파일
올영리뷰_토너_final.csv

주요 컬럼 (가정)
컬럼	설명
review_id	리뷰 ID
review_date	리뷰 작성일
overall_sentiment	전체 감성
absorption_sentiment	흡수 평가
finish_sentiment	마무리감
moisture_sentiment	보습
texture_value	제형
irritation_value	자극
scent_sentiment	향
skin_type	피부타입
purchase_type	구매유형
one_line_summary	요약
2️⃣ 핵심 분석 프레임 (아이디어 결합 구조)
🎯 결합 원칙

[속성 + 감성 + 구매/피부타입] × 월별 변화

3️⃣ 분석 아이디어 → 데이터 검증 설계 (10개)
🔹 IDEA 1

“흡수력은 재구매의 핵심이며, 여름에 더 중요해진다”

분석 조건

absorption_sentiment = POSITIVE

purchase_type = 재구매

월별 비율 변화

검증 지표

월별 재구매 리뷰 중 흡수 긍정 비율

6~8월 증감 확인

🔹 IDEA 2

“점성 제형은 가을·겨울에만 긍정으로 인식된다”

분석 조건

texture_value IN ('점성','쫀쫀')

월별 overall_sentiment

검증

9월 이후 긍정 비율 증가 여부

🔹 IDEA 3

“보습 만족은 줄어도 불만은 여름에 증가한다”

분석 조건

moisture_sentiment

overall_sentiment = NEGATIVE

검증

여름철 ‘보습 부족’ 부정 증가 여부

🔹 IDEA 4

“산뜻함 선호 증가와 보습 불만이 동시에 발생한다”

분석

finish_sentiment = 산뜻

moisture_sentiment = NEGATIVE

검증

동일 월에 동시 증가 여부

🔹 IDEA 5

“향은 계절 무관, 특정 월에만 이슈로 터진다”

분석

scent_sentiment = NEGATIVE

월별 급증 구간 탐색

🔹 IDEA 6

“무난한 평가는 신규 유입기에서 증가한다”

분석

one_line_summary LIKE '%무난%'

purchase_type = 첫구매

월별 비율

🔹 IDEA 7

“지성 피부는 여름에 마무리에 민감해진다”

분석

skin_type = 지성

finish_sentiment = NEGATIVE

월별 변화

🔹 IDEA 8

“자극 이슈는 특정 월에 집중적으로 발생한다”

분석

irritation_value != '없음'

월별 spike 탐지

🔹 IDEA 9

“가성비 평가는 불만을 완충한다”

분석

one_line_summary LIKE '%가성비%'

overall_sentiment 분포

월별 비교

🔹 IDEA 10

“재구매 리뷰는 계절 영향이 작다”

분석

purchase_type = 재구매

월별 속성 변화 표준편차 비교
